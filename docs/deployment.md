# Deployment

Uriel v1 is a local-first NixOS worker. It does not require a hosted
coordination service. The worker owns job state, artifacts, Git worktrees, and
QA evidence on the NixOS host.

## 1. Install The Worker

```nix
{
  services.uriel-worker = {
    enable = true;
    environmentFiles = [
      "/run/secrets/uriel-worker.env"
    ];
  };
}
```

The environment file can be generated by `sops-nix`, `agenix`, or another
NixOS secret mechanism:

```bash
URIEL_WORKER_TOKEN=...
GH_TOKEN=...
OPENCODE_MODEL=...
URIEL_ADAPTER_REPO_BOOTSTRAP=direnv
```

Issue tracker, chat, and ingress integrations are optional adapters. Add their
environment variables only when the deployment enables those adapters.

Deploy the host:

```bash
sudo nixos-rebuild switch --flake .#uriel-worker
```

## 2. Submit Jobs

From the host or a trusted network path:

```bash
export URIEL_WORKER_URL=http://127.0.0.1:8788
export URIEL_WORKER_TOKEN=...

nix run github:zestsystem/uriel#urielctl -- submit \
  --repo https://github.com/zestsystem/uriel.git \
  --prompt "Run a smoke test and report status" \
  --qa browser
```

Use SSH port forwarding, Tailscale, WireGuard, or your preferred reverse proxy
if you need to submit from another machine.

## 3. Android Host Requirements

Use an x86_64 NixOS host with KVM available:

```bash
test -e /dev/kvm
ls -l /dev/kvm
```

The NixOS module adds the `uriel` user to `kvm`, `video`, `render`, and
`adbusers`. If no emulator is booted, Android QA is skipped with a diagnostic
instead of failing the entire job.

## 4. Optional Ingress

Keep ingress outside Uriel core. Good options:

- Tailscale or WireGuard for private API access
- Caddy/nginx for HTTPS and auth
- Hermes gateway for Discord/Slack/Telegram style orchestration
- Any other adapter that can call the local worker API
