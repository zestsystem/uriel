# Architecture

Uriel v1 is a local-first NixOS worker framework. Its core dependency is Nix,
not any hosted coordination service.

## Worker

The NixOS worker runs as a systemd service and owns state under
`/var/lib/uriel`:

- `jobs`: JSON job records and event history
- `repos`: bare Git repository caches
- `worktrees`: one fresh worktree per job
- `artifacts`: logs, transcripts, screenshots, traces, and videos

For every job, the worker:

1. Fetches latest `origin/main`.
2. Creates a fresh branch and worktree from `origin/main`.
3. Reads `AGENTS.md` if present.
4. Evaluates `nix flake show --json --no-write-lock-file` when `flake.nix`
   exists.
5. Lists `just` recipes when a `justfile` exists.
6. Runs profile adapter setup such as `direnv allow`.
7. Runs the selected harness, OpenCode by default.
8. Runs requested browser and/or Android QA.
9. Commits changes, pushes the branch, and creates a draft PR with `gh`.

## Nix-First Boundary

Uriel does not attempt to infer each project by package manager alone. It uses
Nix surfaces first:

- `flake.nix` for tools and shells
- `.envrc`/direnv for environment activation
- repo-local `justfile` for commands
- `AGENTS.md` for agent instructions

This is the piece missing from generic remote coding workers.

## Orchestration

Core orchestration is local HTTP:

- `POST /jobs`
- `GET /jobs`
- `GET /jobs/:id`
- `GET /jobs/:id/events`
- `GET /jobs/:id/artifacts/:name`

External orchestration should be an adapter outside core. Hermes, OpenCode,
Discord bots, Linear webhooks, Slack bots, Tailscale, or a reverse proxy can
all call the worker API without changing Uriel's NixOS execution model.

## Profiles And Adapters

Uriel core is organization-agnostic. It treats a profile as an opaque adopter
identifier such as `acme/mobile` or `platform/web`.

Profile behavior lives behind adapters:

- Harness adapter: runs OpenCode, Hermes, or another coding agent.
- Issue tracker adapter: creates or updates external issue records.
- Repo bootstrap adapter: runs environment activation such as `direnv allow`.
- QA adapter: captures browser and Android evidence.
- Artifact adapter: stores logs, traces, screenshots, and recordings.

Secrets are deliberately outside profile runtime behavior. On NixOS, the host
injects worker credentials through systemd environment files generated by
Nix-native secret tooling.
